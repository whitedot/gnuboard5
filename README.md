# Member-Only Refactor

## 1. 프로젝트 개요
이 프로젝트는 비개발자 출신 지운아빠가 개인적으로 진행하는 사이드 프로젝트입니다.

출발점은 그누보드5(G5) 및 영카트 기반 시스템을 AI가 다루기 쉬운 방향으로 정리해 보자는 문제의식이었습니다. 다만 현재 진행 방식은 전체 기능을 고르게 현대화하는 접근보다, 실제 운영에 필요한 범위를 먼저 줄이고 그 안에서 구조와 UI를 다듬는 방식으로 바뀌었습니다.

현재 저장소는 회원 전용 운영에 필요한 최소 범위만 남기도록 기존 G5 기반 코드를 정리한 결과물에 가깝습니다. 핵심 목표는 다음 두 가지입니다.

- 운영 범위를 계정, 인증, 관리자 회원 관리 중심으로 단순화
- 사람이 직접 유지보수하고 AI 에이전트와 함께 다루기 쉬운 구조로 공용 라이브러리와 관리자 화면을 정리

## 2. 정리 방향

현재 기준의 1단계는 "회원 운영 중심 경량화"입니다. 이 단계에서는 기능을 넓히기보다 유지 범위를 좁히고, 남겨진 화면과 공용 코드를 정리해 작업 단위를 단순하게 만드는 데 초점을 두고 있습니다.

- 회원, 인증, 관리자 회원 관리에 직접 관련된 흐름만 유지
- 게시판, 쇼핑몰, 주문/결제, 소셜, SMS, 기타 부가 관리 도구처럼 현재 운영 범위에서 벗어난 영역은 제거 또는 축소
- 단, 회원가입/비밀번호 찾기/회원정보 보강에 필요한 본인인증 연동 모듈(KG이니시스 간편인증, NHN KCP 휴대폰 본인확인)은 유지
- 공용 진입점과 공용 라이브러리를 다시 나누어 문맥 탐색 비용을 낮춤
- 관리자 화면의 마크업, 스타일, 스크립트 규칙을 정리해 화면별 편차를 줄임
- 제거 대상 기능에 딸려 있던 런타임, 설치, 문서, 정적 자산 의존성을 함께 축소
- 레거시 프론트엔드 의존성을 줄여 수정 범위를 예측하기 쉬운 상태로 정리

## 3. 주요 변경

1단계에서 실제로 진행한 주요 변경은 다음과 같습니다.

- `lib/common.lib.php` 계열 기능을 용도별 파일로 분리
- 회원/인증 중심 URL 흐름 정리 및 호환 레이어 최소화
- 관리자 설정/회원 관리 화면 리팩토링 및 UI 규칙 통일
- Tailwind CSS v4 기반 공용 스타일 체계 도입 및 관리자 화면 정비
- jQuery 런타임 의존 제거 및 관련 보조 스크립트 정리
- 제거 대상 기능의 설치 스키마, 보조 자산, 문서, 정적 파일 축소
- 업스트림 동기화 범위를 회원/인증/관리자 보안 패치 중심으로 재정의

## 4. 개발 환경 및 크레딧

- **기획 및 리서치 (Planning Environment)**: Gemini Deep Research
- **개발 환경 (Development Environment)**: Visual Studio Code, Codex (Windows)
- **사용 모델 (AI Models Used)**:
    - Gemini 3 Pro
    - Codex 5.4

## 5. 향후 계획

향후 계획은 아직 확정하지 않았습니다.

현재로서는 AI 전용 주석 전략을 본격 도입하는 방향보다, 사용자 화면에서 바로 사용할 수 있는 기본 디자인 제공 쪽에 무게를 둘 가능성이 있습니다. 다만 이 부분은 아직 구체적인 범위와 방식이 정해지지 않았으며, 실제 진행 방향은 이후 정리 기준과 운영 목적에 따라 달라질 수 있습니다.

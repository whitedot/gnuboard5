# Member-Only Follow-Up Tasks

## 기준 상태

- 기준 브랜치: `codex/legacy-comment-cleanup-2026-04-22`
- 기준 시점: 2026-04-23
- 현재 정책:
  - 삭제/탈퇴 시 `mb_id`와 상태 정보만 유지한다.
  - 이름, 닉네임, 이메일, 휴대폰, 생년월일, IP, 인증 이력, 동의 로그, 임시 인증값은 비식별 처리 또는 삭제한다.
  - 관리자 UI에서는 탈퇴 회원 `mb_id`를 일부 마스킹해서 표시한다.

## 우선순위

### 1. 관리자 노출 최소화 추가 정리

- 목표:
  - 내부 식별은 유지하되 관리자 화면에서 불필요한 원문 노출을 더 줄인다.
- 할 일:
  - 탈퇴 회원 상세 화면에서 추가 노출 필드가 없는지 재검사
  - 관리자 대시보드, 검색 결과, 로그성 문구에서 탈퇴 계정 `mb_id` 원문 노출 여부 재검사
  - 필요하면 장기적으로 `mb_no` 기반 관리자 라우팅 전환 검토
- 완료 기준:
  - 일반적인 관리자 화면에서 탈퇴 회원 원문 `mb_id`가 노출되지 않는다.

## 제외 범위

- 로컬 QA 재현 절차 문서화는 이번 저장소 후속 작업 범위에서 제외한다.
- 회원 흐름 스모크 자동화는 개별 처리 예정이므로 이 문서의 추적 대상에서 제외한다.

## 보류 판단

- `mb_id`를 인코딩/복호화하는 방식은 현재 채택하지 않는다.
- 이유:
  - 보호 효과 대비 복잡도가 크다.
  - 현재 코드베이스는 `mb_id`를 직접 키로 사용하는 경로가 많다.
  - 현 단계에서는 `mb_id` 내부 보존 + UI 마스킹이 더 실효적이다.

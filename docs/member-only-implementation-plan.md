# Member-Only Implementation Plan

## 현재 상태

- 계정, 인증, 관리자 회원 관리 흐름만 유지한다.
- 제거 대상이던 커뮤니티/상거래, 포인트, SMS, 소셜 로그인, 관리자 보조 기능 정리는 끝났다.
- 공개 페이지 주입성 설정(`cf_analytics`, `cf_add_meta`, `cf_add_script`)과 사용자 디바이스 전환 UI도 제거했다.
- 남은 일은 실제 운영 흐름 검증, 결과 기록, 그리고 홈페이지/전화번호/주소를 최소 회원정보 범위에 계속 둘지 최종 확정하는 것이다.

## 완료 기준

1. 회원 흐름이 `member/` 기준으로 끝까지 동작한다.
2. 신규 설치가 현재 유지 범위에 필요한 스키마만 만든다.
3. 관리자 설정/메일/권한 관리가 제거된 기능 테이블을 참조하지 않는다.
4. BBS, SHOP, 콘텐츠 전용 엔트리포인트와 업그레이드 파일이 저장소에 남아 있지 않다.

## 검증 순서

1. 회원가입
2. 로그인 / 로그아웃
3. 이메일 인증
4. 비밀번호 재설정
5. 회원정보 수정
6. 회원탈퇴
7. 관리자 회원 CRUD

## 세션 재개 체크포인트

1. 현재 브랜치 확인
2. 워킹트리 확인
3. 홈페이지/전화번호/주소 유지 여부 확인
4. 수동 QA 실행
5. 결과 기록 후 필요한 보정 반영

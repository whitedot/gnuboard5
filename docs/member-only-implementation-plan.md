# Member-Only Implementation Plan

## 현재 상태

- 계정, 인증, 관리자 회원 관리 흐름만 유지한다.
- 제거 대상이던 커뮤니티/상거래, 포인트, SMS 정리는 끝났고, 회원/인증 최소 스키마와 레거시 가드 코드 마감 단계다.
- 남은 일은 실제 운영 흐름 검증, 결과 기록, member-only에서 숨김 처리된 비핵심 관리자 설정의 제거 여부 확정이다.

## 완료 기준

1. 회원 흐름이 `member/` 기준으로 끝까지 동작한다.
2. 신규 설치가 현재 유지 범위에 필요한 스키마만 만든다.
3. 관리자 설정/메일/권한 관리가 제거된 기능 테이블을 참조하지 않는다.
4. BBS, SHOP, 콘텐츠 전용 엔트리포인트와 업그레이드 파일이 저장소에 남아 있지 않다.

## 검증 순서

1. 회원가입
2. 로그인 / 로그아웃
3. 이메일 인증
4. 비밀번호 재설정
5. 회원정보 수정
6. 회원탈퇴
7. 관리자 회원 CRUD
8. 소셜 로그인
## 세션 재개 체크포인트

1. 현재 브랜치 확인
2. 워킹트리 확인
3. 수동 QA 실행
4. 결과 기록 후 필요한 보정 반영

# Member-Only Scope

## 목적

이 저장소는 계정, 인증, 관리자 회원 관리에 필요한 최소 범위를 유지한다.

## 유지 범위

- 회원가입
- 로그인 / 로그아웃
- 비밀번호 찾기 / 재설정
- 이메일 인증
- 본인인증 갱신
- 회원정보 조회 / 수정
- 회원탈퇴
- 관리자 회원 CRUD
- 정보공개 동의, 이메일 수신 동의, 마케팅 동의와 동의 이력 관리

## 제외 범위

- 커뮤니티 게시물 작성/조회/검색 계열 전부
- 상품, 주문, 결제 계열 전부
- 다만 회원 인증 흐름에 필요한 본인확인 연동은 예외로 유지
  KG이니시스 간편인증, NHN KCP 휴대폰 본인확인, 인증 시도/이력 관리
- 소셜 로그인, 소셜 계정 연동 전부
- 공개 프로필, 회원간 쪽지, 스크랩, 설문, 자주 묻는 질문 계열 전부
- 홈페이지, 일반 전화번호, 주소 입력/저장 계열 전부
- 현재 접속자 집계, 사용자 디바이스 전환, 포인트, SMS, 회원 대상 운영 메일 계열 전부

## 현재 기준 파일

- `common.php`
- `config.php`
- `member/`
- `adm/member_*`
- `lib/common.*.lib.php`
- `lib/uri.lib.php`
- `lib/URI/uri.class.php`

## 운영 원칙

1. 회원 기능의 구현 원천은 `member/`에 둔다.
2. BBS, SHOP 전용 엔트리포인트는 스텁으로 남기지 않고 삭제한다.
3. 운영용 스타일 출력만 유지하고, 문서 전용 UI 레퍼런스와 빌드는 저장소 범위에서 제외한다.
4. 제거된 기능을 되살리는 호환 레이어는 추가하지 않는다.
5. 신규 설치 스키마는 현재 유지 범위와 법적 동의 기록에 필요한 테이블만 생성한다.
6. 유지 범위 파일 안에 숨어 있는 BBS, SHOP 파생 설정 참조도 허용하지 않는다.

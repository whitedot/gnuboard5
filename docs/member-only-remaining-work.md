# Member-Only Remaining Work

## 목적

회원 전용 구조 전환 이후 남은 정리 작업을 우선순위와 성격별로 고정한다.

## 현재 상태 요약

- `member/` 기준으로 로그인, 가입, 인증, 비밀번호 재설정 흐름을 분리했다.
- `SHOP` 텍스트 자산과 주요 `BBS` 기능은 제거했다.
- 관리자와 기본 테마도 회원 전용 기준으로 단순화했다.
- 남은 이슈는 주로 바이너리 잔재, 미사용 공용 함수, 조건부 회원 기능 정리다.

## 우선순위 1

### 1. `shop/` 바이너리 잔재 물리 삭제

현재 텍스트 파일은 제거됐지만 아래 바이너리/이미지 파일은 디렉터리에 남아 있다.

- `shop/kcp/bin/*`
- `shop/kcp/img/*`
- `shop/mail/img/*`

작업:

- 디렉터리 전체 삭제
- 배포 스크립트나 CI에서 `shop/` 경로를 더 이상 기대하지 않는지 확인

완료 기준:

- `shop/` 디렉터리가 완전히 비어 있거나 제거됨

### 2. 회원 전용 기능 범위 최종 확정

현재 코드상 살아 있을 수 있는 조건부 회원 기능:

- 쪽지
- 포인트
- 공개 프로필

작업:

- 유지 여부 결정
- 제거 결정 시 엔트리포인트, 스킨, 관리자 메뉴, 공용 헬퍼 동시 정리

완료 기준:

- 회원 서비스에서 제공하는 기능 목록이 문서와 코드에 일치

## 우선순위 2

### 3. 미사용 공용 함수 정리

후보 파일:

- `lib/common.util.lib.php`
- `lib/get_data.lib.php`
- `lib/common.session.lib.php`
- `lib/outlogin.lib.php`

정리 후보:

- autosave 관련 함수
- scrap 관련 함수
- QA 관련 조회 함수
- shop 관련 보조 분기

완료 기준:

- 삭제된 기능을 위한 함수가 더 이상 런타임에서 호출되지 않음

### 4. 관리자 업그레이드/설정 화면 추가 축소

후보 파일:

- `adm/dbupgrade.php`
- `adm/newwinlist.php`
- `adm/config_form_update.php`

작업:

- 더 이상 쓰지 않는 게시판/쇼핑/통계 관련 업그레이드 분기 제거
- 회원 전용 서비스에서 의미 없는 설정 항목 정리

완료 기준:

- 관리자에서 삭제된 기능 관련 문구, 버튼, 저장 로직이 남지 않음

## 우선순위 3

### 5. 라우팅/상수 정리

현재는 호환을 위해 `G5_BBS_*` 경로와 리다이렉트를 일부 유지하고 있다.

작업:

- `member/` 기준 상수 사용 범위 확대
- 구 `bbs` 회원 리다이렉트 유지 기간 결정
- 필요 시 `bbs` 디렉터리를 최소 호환 계층으로 축소

완료 기준:

- 회원 기능 경로가 일관되게 `member/`를 기준으로 동작

### 6. 테스트 체크리스트 확정

필수 확인 흐름:

- 회원가입
- 로그인 / 로그아웃
- 이메일 인증
- 비밀번호 찾기 / 재설정
- 회원정보 수정
- 회원탈퇴
- 관리자 회원 생성 / 수정 / 삭제

완료 기준:

- 수동 검증 항목 또는 자동화 가능한 테스트 항목 문서화

## 보류 항목

- `bbs` 경로 완전 제거
- DB 스키마 차원의 불필요 테이블 삭제
- 레거시 상수/훅 호환성 완전 제거

위 항목은 운영 중단이나 데이터 마이그레이션과 연결될 수 있으므로 별도 단계로 분리한다.

# Member-Only Remaining Work

## 목적

member-only 전환 구현은 끝났고, 이제 남은 일은 실제 운영 흐름이 깨지지 않았는지 확인하는 것이다.

이 문서는 다음 세션이나 QA 담당자가 그대로 따라할 수 있는 최종 수동 테스트 체크리스트를 고정한다.

## 현재 상태 요약

- 유지 범위는 `member/` 중심의 계정, 인증, 회원관리 기능이다.
- 게시판, 쇼핑몰, 쪽지, 포인트, 공개 프로필 진입점은 종료되었거나 member-only 가드로 차단되었다.
- `bbs`의 회원 처리 파일은 대부분 `member/` 구현을 호출하는 호환 레이어로 축소되었다.
- 공용 라이브러리와 URL 헬퍼는 member-only에서 게시판 전용 링크를 새로 만들지 않도록 정리되었다.
- 관리자 메뉴/설정 저장은 종료된 기능 링크와 값을 최대한 남기지 않도록 정리되었다.
- 남은 실작업은 아래 체크리스트를 실행하고 결과를 기록하는 것이다.

## 구현 완료 범위

- `shop/` 자산 및 설치 시 쇼핑몰 초기화 경로 제거
- `bbs/memo*.php`, `bbs/point.php`, `bbs/profile.php`, `bbs/password*.php` 비활성화
- 게시판 링크 / 첨부 / 썸네일 URL 생성 차단
- `alert.php`, `alert_close.php`, `confirm.php` 공용 템플릿 루트 이동
- member-only 기준 `content` rewrite만 유지하도록 URL 정리
- 관리자 메뉴 링크 치환 및 무의미 설정 저장값 중립화
- `orderupgrade.php` 차단
- 종료 기능용 `extend` 훅과 미사용 `js` 자산 정리

## 최종 수동 테스트 체크리스트

### 사전 준비

- 테스트 브랜치는 `codex/member-only-refactor` 기준으로 맞춘다.
- DB는 member-only 전환 스키마와 관리자 계정이 준비된 상태여야 한다.
- 메일 발송 확인이 가능해야 하며, 소셜 로그인은 최소 1개 제공자 설정이 필요하다.
- 브라우저는 일반 창 1개와 시크릿 창 1개를 준비하면 검증이 빠르다.

### 1. 회원가입

절차:

1. 비로그인 상태에서 회원가입 화면으로 진입한다.
2. 약관 동의, 가입 폼 입력, 본인인증 또는 캡차가 있다면 함께 완료한다.
3. 가입 완료 후 안내 화면 또는 로그인 유도 화면으로 이동하는지 확인한다.

기대 결과:

- 가입 경로가 `member/` 기준으로 동작한다.
- 가입 중 게시판, 쇼핑몰, 포인트, 쿠폰 관련 문구나 처리 흔적이 나타나지 않는다.
- 신규 회원 레코드가 관리자 회원 목록에서 바로 확인된다.

### 2. 로그인 / 로그아웃

절차:

1. 신규 계정 또는 기존 계정으로 로그인한다.
2. 로그인 후 헤더, 마이페이지, 정보수정 링크가 정상 노출되는지 확인한다.
3. 로그아웃 후 보호 페이지 접근 시 다시 로그인 화면으로 이동하는지 확인한다.

기대 결과:

- 로그인/로그아웃 모두 `member/` 경로 기준으로 동작한다.
- 세션 처리 후 종료된 게시판/쇼핑몰 페이지로 튀지 않는다.
- 로그아웃 이후 세션이 남지 않는다.

### 3. 이메일 인증

절차:

1. 이메일 인증이 필요한 계정으로 로그인하거나 가입 직후 인증 메일을 발송한다.
2. 발송된 메일의 인증 링크를 연다.
3. 인증 완료 후 상태 변경과 후속 이동 경로를 확인한다.

기대 결과:

- 인증 메일 링크가 유효하고, 인증 완료 후 계정 상태가 정상 반영된다.
- 인증 완료 화면이 게시판 의존 없이 정상 렌더링된다.
- 인증 실패 또는 재사용 링크 처리도 사용자에게 명확히 안내된다.

### 4. 비밀번호 찾기 / 재설정

절차:

1. 로그인하지 않은 상태에서 비밀번호 찾기 화면으로 진입한다.
2. 계정 식별 정보를 입력해 재설정 메일을 발송한다.
3. 메일의 재설정 링크로 들어가 새 비밀번호를 저장한다.
4. 새 비밀번호로 다시 로그인한다.

기대 결과:

- 찾기, 메일 발송, 재설정, 재로그인 흐름이 모두 끊기지 않는다.
- 링크가 `member/` 기준으로 연결되며 `bbs/password.php` 계열로 빠지지 않는다.
- 재설정 완료 후 이전 비밀번호는 더 이상 동작하지 않는다.

### 5. 회원정보 수정

절차:

1. 로그인 상태에서 정보수정 화면으로 진입한다.
2. 닉네임, 이메일, 연락처 등 수정 가능한 값을 변경한다.
3. 비밀번호 변경이 가능하면 함께 수행하고 다시 로그인해 확인한다.

기대 결과:

- 저장 후 변경값이 즉시 반영된다.
- member-only에서 의미 없는 게시판/쇼핑몰 설정 필드가 노출되지 않는다.
- 비밀번호 변경이 포함된 경우 세션 재인증 흐름이 정상 동작한다.

### 6. 회원탈퇴

절차:

1. 테스트 계정으로 로그인한다.
2. 회원탈퇴 화면에서 안내 문구와 확인 절차를 확인한다.
3. 탈퇴 완료 후 동일 계정 로그인 가능 여부와 관리자 표시 상태를 확인한다.

기대 결과:

- 탈퇴 후 세션이 종료된다.
- 탈퇴 계정은 정책에 맞는 상태로 처리된다.
- 탈퇴 완료 화면이나 후속 이동 경로가 종료된 기능 페이지를 참조하지 않는다.

### 7. 관리자 회원 생성 / 수정 / 삭제

절차:

1. 관리자에서 회원 생성 기능으로 신규 회원을 추가한다.
2. 생성한 회원의 권한, 상태, 기본 프로필 값을 수정한다.
3. 필요 시 관리자에서 탈퇴 또는 삭제 처리까지 확인한다.

기대 결과:

- 관리자 회원 CRUD가 정상 동작한다.
- 회원 저장 시 포인트, 쪽지, 쇼핑몰, 게시판 의존 기능 때문에 오류가 나지 않는다.
- 메뉴 링크 저장 시 종료된 기능 URL은 홈으로 치환되거나 저장되지 않는다.

### 8. 소셜 로그인 연동 계정 처리

절차:

1. 지원하는 소셜 로그인 제공자로 신규 가입 또는 기존 계정 연동을 시도한다.
2. 최초 연동 후 회원정보 화면과 로그인 상태를 확인한다.
3. 로그아웃 후 같은 제공자로 재로그인해 기존 계정이 재사용되는지 확인한다.

기대 결과:

- 소셜 로그인과 연동 계정 매핑이 정상 동작한다.
- 연동 후 회원 흐름이 `member/` 기준으로 유지된다.
- 소셜 로그인 완료 후 종료된 게시판/쇼핑몰 경로로 이동하지 않는다.

## 실행 후 기록할 항목

- 테스트 일시
- 사용 브랜치와 커밋 해시
- 실패한 절차 번호
- 실제 결과와 기대 결과 차이
- 재현 조건과 관련 스크린샷 또는 에러 로그

## 보류 항목

- `bbs` 경로 완전 제거
- DB 테이블 / 컬럼 물리 삭제
- 레거시 상수 / 훅 호환성 완전 제거

위 항목은 운영 데이터 이전이나 하위 호환 종료와 연결되므로 별도 단계로 분리한다.

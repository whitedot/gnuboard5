# Member-Only Remaining Work

## 목적

회원 전용 구조 전환 이후 남은 정리 작업만 추려서 고정한다.

## 현재 상태 요약

- `member/` 기준으로 로그인, 가입, 인증, 비밀번호 재설정 흐름을 분리했다.
- 회원 관련 메일, 소셜 로그인, 본인인증 결과 처리도 `member/` 기준 경로로 정리했다.
- `shop/` 바이너리와 이미지 잔재를 포함한 쇼핑몰 흔적을 정리했고, 신규 설치에서 쇼핑몰 생성도 끊었다.
- 쪽지, 포인트, 공개 프로필은 사용자 진입점과 관리자 노출을 종료했다.
- FAQ 설정, poll 헬퍼 같은 비회원 기능 잔재도 함께 축소했다.

## 완료된 정리

### 1. 회원 전용 범위 확정

- 유지: 계정, 인증, 회원관리
- 종료: 게시판, 쇼핑몰, 쪽지, 포인트, 공개 프로필

### 2. 코드 정리 완료 항목

- `shop/` 바이너리 및 이미지 잔재 삭제
- `bbs/memo*.php`, `bbs/point.php`, `bbs/profile.php` 비활성화 스텁 전환
- 포인트 적립 및 재계산 로직 제거
- 관리자 포인트 메뉴/화면/설정 항목 제거
- 회원가입 후 쇼핑 쿠폰 지급 로직 비활성화
- 설치 화면과 설치 SQL 기본값에서 쇼핑몰 / FAQ 의존 축소
- `theme/basic/head.sub.php` 및 관리자/플러그인 회원 URL을 `member/` 기준으로 통일
- 미사용 조회 함수와 회원 전용과 무관한 헬퍼 정리

## 남은 작업

### 1. 관리자 업그레이드/설정 화면 추가 축소

후보 파일:

- `adm/dbupgrade.php`
- `adm/newwinform.php`
- `adm/thumbnail_file_delete.php`

작업:

- `G5_MEMBER_ONLY` 조건으로만 남겨둔 쇼핑/게시판 관련 분기를 실제 제거 가능한지 점검
- 회원 전용 서비스에서 의미 없는 설정 문구와 업그레이드 항목 추가 축소

완료 기준:

- 관리자에서 삭제된 기능 관련 문구, 버튼, 저장 로직이 남지 않음

### 2. 라우팅/상수 정리

현재 회원 인증/가입 계열은 대부분 `member/` 기준으로 바뀌었지만, 게시판/공용 헬퍼 레벨에서는 `G5_BBS_*`가 남아 있다.

작업:

- 게시판 전용이 아닌 공용 JS/테마 변수의 `G5_BBS_*` 잔재 점검
- `plugin/sns`, `plugin/syndi`, `lib/uri.lib.php` 등 게시판 결합부의 정리 시점 결정
- 구 `bbs` 회원 리다이렉트 유지 기간 결정

완료 기준:

- 회원 기능 경로가 일관되게 `member/`를 기준으로 동작하고, `bbs`는 최소 호환 레이어만 남음

### 3. 테스트 체크리스트 확정

필수 확인 흐름:

- 회원가입
- 로그인 / 로그아웃
- 이메일 인증
- 비밀번호 찾기 / 재설정
- 회원정보 수정
- 회원탈퇴
- 관리자 회원 생성 / 수정 / 삭제

완료 기준:

- 수동 검증 항목 또는 자동화 가능한 테스트 항목 문서화

## 보류 항목

- `bbs` 경로 완전 제거
- DB 스키마 차원의 불필요 테이블과 컬럼 정리
- 레거시 상수/훅 호환성 완전 제거

위 항목은 운영 중단이나 데이터 마이그레이션과 연결될 수 있으므로 별도 단계로 분리한다.
